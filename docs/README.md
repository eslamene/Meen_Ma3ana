## üìù Documentation Standards

### File Naming
- Use `UPPERCASE_WITH_UNDERSCORES.md` for main documentation files
- Use `lowercase-with-hyphens.md` for guides and references
- Keep filenames descriptive and consistent

### Structure
- Each major section should have a README.md with an overview
- Related documentation should be grouped in subdirectories
- Remove outdated documentation that is no longer relevant

### Content Guidelines
- Start with an overview/description
- Include prerequisites when applicable
- Provide step-by-step instructions
- Include troubleshooting sections for complex topics
- Link to related documentation
